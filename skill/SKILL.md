# GuardRail Hermes Skill

GuardRail is a Hermes Agent skill that provides **pre-send content scanning** for agent workflows. It acts as a safety guard that intercepts tool outputs and AI-generated messages before they are delivered to the user, enforcing configurable content policies.

## Installation

Place this skill in your Hermes skills directory (e.g., `~/.hermes/skills/guardrail/`) and ensure the following files are present:

- `skill.yaml` — Skill manifest
- `SKILL.md` — This documentation file
- `call.py` — Main skill logic (scan/block/report)

## Usage

Load the skill in your Hermes Agent session:

```
skill_load(name='guardrail')
```

Once loaded, GuardRail hooks into the agent's pre-send pipeline. Every message generated by the agent is scanned before being sent to the user.

## Configuration

GuardRail behavior is controlled via the `skill.yaml` manifest:

```yaml
name: guardrail
category: security
config:
  enabled: true
  rules:
    block_pii: true
    block_code_injection: true
    block_dangerous_commands: true
  log_violations: true
```

### Options

| Option              | Type    | Default | Description                                    |
|---------------------|---------|---------|------------------------------------------------|
| `enabled`           | boolean | `true`  | Master switch for GuardRail scanning           |
| `block_pii`         | boolean | `true`  | Block messages containing PII                  |
| `block_code_injection` | boolean | `true` | Block messages with code injection patterns |
| `block_dangerous_commands` | boolean | `true` | Block messages containing dangerous shell commands |
| `log_violations`    | boolean | `true`  | Log blocked messages to the violation log      |

## How It Works

1. The Hermes Agent generates a response or prepares a tool output message.
2. Before sending, GuardRail intercepts the message and runs configured content checks.
3. If all checks pass, the message is delivered normally.
4. If a violation is detected, the message is blocked, logged, and an optional alert is raised.

## Scanning Rules

GuardRail applies the following rules in order:

- **PII Detection**: Scans for email addresses, phone numbers, SSNs, credit card numbers, and other personally identifiable information.
- **Code Injection Detection**: Flags patterns that resemble SQL injection, command injection, or other injection attacks.
- **Dangerous Command Detection**: Blocks messages containing known dangerous shell commands or system-modifying operations.

## Integration with Agent Workflows

GuardRail can be combined with other skills for layered security:

```python
# Load security skills
skill_load(name='guardrail')
skill_load(name='audit_log')
```

## Violation Log

When `log_violations` is enabled, violations are written to `~/.hermes/logs/guardrail_violations.log` with the following format:

```
[TIMESTAMP] BLOCKED — rule=<rule_name> — message_preview=<first_200_chars>
```

## Troubleshooting

- **GuardRail not blocking content**: Verify `enabled: true` in `skill.yaml`.
- **False positives**: Adjust rule sensitivity or disable specific rules in the config.
- **Skill not loading**: Ensure all required files (`skill.yaml`, `call.py`) are present in the skill directory.
